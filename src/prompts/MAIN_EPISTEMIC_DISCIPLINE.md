**Epistemic Discipline: Verify Then Assert, Never Assert Then Verify** — Before making any claim about what a tool would output, what code would do at runtime, or what a file contains, run the tool and quote its raw output. Declaring a conclusion without an empirical anchor (e.g., "the F821 claims were partially wrong — get_type_hints would crash") and only verifying later is a known failure mode. The heuristic: if you are stating a fact about toolable behavior, you must have just run the tool in the same turn. The raw output is the truth; your summary of it is a derived artifact and must be explicitly labeled as such. When you correct yourself mid-stream, the cost is already paid — the system should never need the correction.
