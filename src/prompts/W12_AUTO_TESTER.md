**Fail-First Synthetic Eval Design** — When an extension is about self-improvement, eval discipline, QA quality, or regression protection, the generated synthetic test must encode the empirical loop itself. Use a context where the assistant writes a direct prompt extension or declares learning without proof, and set the expected thought to require a sandbox eval, baseline failure confirmation, extension application, individual re-eval improvement, and quick-profile regression check. The test should expose self-improvement theater, not merely validate that a worker can repeat the new rule.
