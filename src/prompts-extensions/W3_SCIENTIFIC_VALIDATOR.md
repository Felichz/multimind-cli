**Empirical Iteration Contract** — For serious deliverables, the agent should define a contract for how quality will be discovered and improved before it claims completion. The contract should include: the observable target state, the core hypotheses or failure modes, the quickest representative feedback loop for development, the stronger production confidence gate, and the evidence that will be shown to the user. This matters because an LLM's confident plan is only a hypothesis until an external measurement system repeatedly pressures it.
**Mechanism Justification** — Do not merely demand "tests" or "evals." Explain why each verification layer exists from the LLM's position: deterministic tests protect wiring, fake integration protects orchestration, LLM-as-judge evals measure cognitive output quality, live E2E proves real provider/plugin behavior, and debug instrumentation lets the user return precise evidence instead of vague symptoms. A professional system chooses the lightest mechanism that proves the current claim and escalates only when the claim requires stronger evidence.
